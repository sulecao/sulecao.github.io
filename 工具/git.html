# git

设置用户名称与邮件地址。 每一个 Git 的提交都会使用这些信息，并且会写入到你的每一次提交中，不可更改：

$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com
git config <key>： 来检查 Git 的某一项配置 $ git config user.name
git clone [url]将仓库文件clone到本地并建立链接（Git 支持多种数据传输协议）

git init //初始化
rm  -rf .git //删除文件夹
rm 文件名 //删除文件
mkdir 文件名 //创建文件夹
cd 目录名 //进入文件夹
cd d: //进入d盘
ls -al//显示目录下所有文件
touch 文件名  //创建文件
cat 文件名	//查看文件内容
vi 文件名 //编辑文件内容   :wq保存退出  :q!不保存退出
git log //查看提交日志

如何把提交的commit还原 比较三个区的区别 工作区 暂存区 版本库（有版本号）

git diff   //工作区和暂存区区别

git diff  '分支名' //工作区和历史区区别

git diff --cached //暂存区和历史区区别

撤销

从暂存区取文件覆盖工作区

git checkout 文件名

git reset HEAD 文件名  //回滚暂存区的版本

git reset --hard 版本号 //回退历史区的版本

git reset --hard HEAD ^ //回退到上一个历史区的版本

查看所有版本（回退版本后看到所有的）

git reflog

查看分支和创建分支

git branch

git branch 分支名

切换分支 git checkout 分支名 git checkout -b 分支名 //创建并切换分支

删除分支 git branch -D 分支名

文件修改后，切换分支时，需要保存文件，可以提交或暂存

git stash 暂存文件

回来后再用pop取出来

git stash pop

合并 分支 git merage 分支名 //将分支合入当前所在分支 git remote add 仓库名 地址 // 仓库名自己取，一般设为origin git push -u 仓库名 分支名 // git remote -v //查看关联的仓库 git remote rm -origin //删除关联仓库origin .gitingore
