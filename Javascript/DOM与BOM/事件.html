# 事件流

包括三个阶段：事件捕获阶段、处于目标阶段和事件冒泡阶段。 首先发生的是事件捕获，为截获事件提供了机会。然后是实际的目标接收到事件。最后一个阶段是冒泡阶段，可以在这个阶段对事件做出响应。

，但是也有特例，如果子节点同时注册冒泡和捕获事件，事件触发会按照注册的顺序执行。

# DOM0 级事件处理程序

无法给对象同一个事件注册多个函数，冒泡触发

var btn = document.getElementById("myBtn"); 
btn.onclick = function(){ 
 alert("Clicked"); 
};

# DOM2级事件处理程序

增加和删除事件：addEventListener()和 removeEventListener()。

接受 3 个参数：事件名、回调函数、布尔值或对象。

布尔值 true，捕获阶段调用事件； false，在冒泡阶段调用事件。默认false

对象可以设置capture，once，passive

addEventListener() //可以同时为一个元素绑定多个响应函数，按照函数顺序从上往下执行，ie8及以下不支持
node.addEventListener("click", function(){ 
 alert(this.id); 
}, false); 
node.addEventListener("click", function(){ 
 alert("Hello world!"); 
}, false); 
//会先显示id再显示Hello world!

// capture 是否捕获
// once 触发一次
// passive 永远不会调用 preventDefault()。如果 listener 仍然调用了这个函数，客户端将会忽略它并抛出一个控制台警告。
node.addEventListener(
  'click',
  event => {console.log('点击了')},
 { capture:false,once:true,passive:true}
)

# 移除事件

通过 addEventListener()添加的事件只能使用 removeEventListener()来移除，匿名函数无法移除。

node.removeEventListener("click", handler, false); //点击后移除了事件

//attachEvent()ie8及以下用，事件参数加要on
btn.addEventListener("onclick",function(){})

//事件不再被分派到其他节点。
event.stopPropagation()
//要阻止特定事件的默认行为，可以使用 preventDefault()方法或者 return false

# 事件的几组重要参数

event.clientX //鼠标的x坐标（相对于浏览器里的内容展示区，不包括书签地址栏等）
event.clientY
event.pageX //鼠标的x坐标（相对于整个页面，没有滚动的情况下和clientX相等）
event.pageY 
event.sccreenX //鼠标的x坐标（相对于整个电脑屏幕）
event.sccreenY 
event.offsetX //鼠标指针位置相对于触发事件的对象的x坐标
event.offsetY 
Element.getBoundingClientRect() //返回元素的大小及其相对于视口的位置。 注意是元素

# 事件委派

将事件统一委派给祖先元素，当后代元素触发时，会一直冒泡到祖先元素，从而通过祖先元素的响应函数来处理事件。比如给ul绑定事件，则子元素li都可以触发事件。可以节省内存，也不需要给子节点注销事件 可以通过event.target获得触发事件的元素
